# 2.2. Módulo Notação UML – Modelagem Dinâmica

## Diagrama de Sequência

### Introdução

O **Diagrama de Sequência** é um dos diagramas dinâmicos da UML (*Unified Modeling Language*) utilizado para representar a interação entre objetos de um sistema de software ao longo do tempo [1]. Esse diagrama permite visualizar a ordem cronológica das mensagens trocadas entre os objetos, evidenciando o fluxo de controle do sistema [1][2]. Em consulta ao material disponibilizado pela professora, foi identificado como o mais relevante na UML, em especial, para fase de projeto de software.

### Metodologia

A construção do Diagrama de Sequência foi orientada pelas interações dos usuários com o sistema, no fluxo de envio de mensagens pelo Telegram (um provider de mensageria), mapeadas a partir das funcionalidades definidas no [backlog do projeto (Decision)](Modelagem/Base/DesignSprint/Decision.md) e na organização estrutural detalhada no [Diagrama de Componentes](Modelagem/2.1.ModelagemEstatica.md#diagrama-de-componentes).
