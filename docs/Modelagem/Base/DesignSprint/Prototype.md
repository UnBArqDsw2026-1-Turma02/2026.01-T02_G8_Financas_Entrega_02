# Prototype 

## Introdução 

A etapa de **Prototype** é a quarta fase do Design Sprint, sucedendo a fase de [Decision](Decision.md). Conforme definido por Jake Knapp [1], esta etapa é dedicada à **construção de um protótipo de alta fidelidade** que simula o produto final, permitindo que as soluções desenvolvidas na etapa anterior sejam testadas com usuários reais.

O protótipo é uma representação visual interativa do projeto que permite que os usuários interajam com o sistema e forneçam feedback[1]. O protótipo desenvolvido serviu como insumo para a [Verificação](Verificacao.md), que avaliou a cobertura das funcionalidades, e para a [Validação](Validacao.md), que conduziu testes de usabilidade com usuários. A rastreabilidade entre as histórias de usuário e as telas do protótipo está documentada na [Matriz de Rastreabilidade](MatrizRastreabilidade.md).

## Metodologia

Foi desenvolvido um protótipo de alta fidelidade[2], utilizando a ferramenta Figma[3], com base no backlog definido na etapa de [Decision](Decision.md). Protótipos de alta fidelidade são representações que se aproximam ao máximo do produto final em termos de design visual, interações e navegação, permitindo uma avaliação mais precisa da experiência do usuário[2]. Para desenvolver o protótipo, foi utilizada a metodologia do Atomic Design, que consiste em criar componentes reutálizáveis organizados em níveis hierárquicos (átomos, moléculas, organismos, templates e páginas)[4].
