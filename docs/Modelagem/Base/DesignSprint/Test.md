# Test

## Introdução

A etapa de **Test** é a quinta e última fase do Design Sprint, conforme proposto por Jake Knapp [1]. Após a construção do protótipo na fase de [Prototype](Prototype.md), esta etapa tem como objetivo **testar e validar** a solução com usuários reais ou por meio de avaliações estruturadas, coletando feedback para identificar pontos fortes e oportunidades de melhoria antes do desenvolvimento [1].

Esta etapa consiste em verificar e validar o protótipo criado na etapa anterior, verificando se ele contempla todas as funcionalidades e fluxos requeridos. Para isso, foram criados dois documentos: um checklist para verificar se o protótipo continha todas as funcionalidades que o artefato [Decision](Decision.md) previa, e um Teste de Usabilidade para validar se o protótipo atendia aos requisitos definidos, presente no artefato [Prototype](Prototype.md). Os artefatos gerados são:

- [Checklist de Verificação do Protótipo](Verificacao.md)
- [Teste de Usabilidade](Validacao.md)
