# Unpack 

## Introdução 

A etapa de **Unpack** é a primeira fase do Design Sprint, conforme proposto por Jake Knapp [1]. Nesta etapa, o time analisa o problema, define o objetivo e reúne o máximo de contexto sobre o desafio. O objetivo é entender o problema e coletar a maior quantidade de insights para depois serem debatidos [1]. Os artefatos gerados nesta fase — Introspecção e Personas — serviram como insumo direto para a etapa de [Sketch](Sketch.md), para a construção do [Backlog](Decision.md) e estão documentados na [Matriz de Rastreabilidade](MatrizRastreabilidade.md).

## Metodologia

As metodologias utilizadas nessa etapa foram a Introspecção e as Personas para entender melhor o problema e o contexto do projeto [2]. Antes da execução das metodologias, foi realizada uma pesquisa sobre o tema do projeto para obter o máximo de informações e ideias possíveis, pesquisando sobre outros aplicativos e sites do mesmo tema — como a planilha de gastos do BTG Pactual [3] e os modelos de gerenciamento financeiro do ClickUp [4] — para obter informações sobre como eles funcionam e quais funcionalidades eles oferecem. 

### Introspecção

A introspecção é uma técnica de elicitação de requisitos em que o analista se coloca na posição do usuário para identificar necessidades e funcionalidades de um sistema [2]. Por esse motivo, foi selecionada essa metodologia para essa etapa do projeto, com o objetivo de obter o máximo de informações e ideias para resolver o problema proposto. Como há apenas um integrante na equipe, a Introspecção foi realizada com base nas [Personas](#personas) definidas abaixo e nas [pesquisas realizadas sobre o tema do projeto](../../README.md#sobre). A partir dessa técnica, foram levantados requisitos que posteriormente alimentaram o [Backlog do Projeto](Decision.md) e os artefatos da etapa de [Sketch](Sketch.md), como o Mapa Mental e o Rich Picture.

### Personas

Personas são representações fictícias de usuários reais, criadas com base em dados e pesquisas para compreender quem é o público-alvo, suas dores, necessidades, comportamentos e contexto de uso [2]. Por serem fundamentais para guiar decisões centradas no usuário, as personas foram selecionadas para essa etapa do projeto. Foram desenvolvidas 4 personas para representar os usuários do projeto, cada uma exercendo um papel central na elaboração da [Introspecção](#introspecção) e na construção do [Rich Picture](Sketch.md#rich-picture) na etapa de Sketch:

## Conclusão

Foram gerados dois artefatos com o objetivo de entender melhor o problema e o contexto do projeto. O primeiro artefato foi a **Introspecção**, que, a partir da perspectiva das personas, permitiu levantar requisitos e funcionalidades para o sistema. O segundo artefato foi o de **Personas**, que representou os perfis de usuário e suas necessidades. Juntos, esses artefatos forneceram uma base sólida para a etapa seguinte — [Sketch](Sketch.md) e para as etapas subsequentes.
