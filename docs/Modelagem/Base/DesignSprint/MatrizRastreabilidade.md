# Matriz de Rastreabilidade

## Introdução 

A matriz de rastreabilidade é uma ferramenta de engenharia de requisitos que auxilia na gestão do projeto, garantindo a rastreabilidade bidirecional entre os requisitos e os artefatos que os originaram[1]. Seu uso permite verificar a cobertura dos requisitos, identificar impactos de mudanças e assegurar que todas as funcionalidades definidas no [Backlog](Decision.md) possuem origem documentada[2].

## Metodologia

Para construir a matriz de rastreabilidade, foi utilizada uma tabela que documenta a origem de cada história de usuário do [Backlog](Decision.md) e os artefatos produzidos na etapa de [Sketch](Sketch.md) que a originaram. A coluna "Protótipo" mapeia cada história às telas correspondentes no [Protótipo](Prototype.md).
